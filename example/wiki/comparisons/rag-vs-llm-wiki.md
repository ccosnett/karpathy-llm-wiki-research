# Comparison: RAG vs LLM Wiki

## RAG

- retrieves source fragments at query time
- synthesizes answers on demand
- often repeats the same synthesis work across questions

## LLM Wiki

- compiles source material into persistent markdown pages
- maintains cross-links and updates over time
- keeps useful syntheses as durable artifacts

## Practical Difference

RAG is mainly a retrieval-time answering pattern. An LLM wiki is a maintenance pattern for building a reusable knowledge layer.
